# YOLO-RGBT 中文操作说明书

本文档说明如何使用本项目训练、验证、推理和导出 RGB + 红外双模态目标检测模型。当前 RGBT 逻辑适用于 YOLOv5、YOLOv8、YOLO11 和 YOLO26 的 RGBT 模型配置。

## 1. 功能说明

本项目的 RGBT 输入由两张图像组成：

- 可见光图像：通常是 RGB/BGR 三通道图像。
- 红外图像：通常是单通道灰度图像。

训练和推理时，程序会自动把二者合成为 4 通道输入：

```text
BGR 3通道 + 红外 T 1通道 = 4通道 RGBT
```

因此你不需要提前把图像合成 4 通道文件，只需要保证可见光图像和红外图像一一对应。

## 2. 推荐数据集目录结构

推荐使用如下目录结构：

```text
YourDataset/
  visible/
    train/
      img001.jpg
      img001.txt
      img002.jpg
      img002.txt
    val/
      img101.jpg
      img101.txt
    test/
      img201.jpg
      img201.txt

  infrared/
    train/
      img001.jpg
      img002.jpg
    val/
      img101.jpg
    test/
      img201.jpg
```

关键要求：

1. `visible` 和 `infrared` 两个目录必须同级。
2. 可见光和红外图像的相对路径、文件名必须一致。
3. 数据集 yaml 中填写可见光路径，例如 `train: visible/train`。
4. 程序会根据 `rgbt_pair: [visible, infrared]` 自动找到红外图。

例如：

```text
YourDataset/visible/train/img001.jpg
```

会自动配对：

```text
YourDataset/infrared/train/img001.jpg
```

## 3. 标签格式

标签使用标准 YOLO txt 格式：

```text
class_id x_center y_center width height
```

坐标必须是归一化坐标，范围为 `0-1`。

示例：

```text
0 0.512 0.433 0.120 0.280
```

推荐把标签放在可见光目录下：

```text
visible/train/img001.jpg
visible/train/img001.txt
```

红外目录下可以不放标签，只要可见光目录下的标签能被 YOLO 正常读取即可。

## 4. 数据集 yaml 写法

可以复制模板：

```text
ultralytics/cfg/datasets/rgbt-template.yaml
```

一个典型配置如下：

```yaml
path: D:/datasets/YourDataset
train: visible/train
val: visible/val
test: visible/test

channels: 4
rgbt: true
rgbt_pair: [visible, infrared]

names:
  0: person
  1: car
  2: bicycle
```

字段说明：

- `path`：数据集根目录。
- `train`：训练集可见光图像目录，相对于 `path`。
- `val`：验证集可见光图像目录，相对于 `path`。
- `test`：测试集可见光图像目录，可选。
- `channels: 4`：告诉模型输入是 4 通道。
- `rgbt: true`：启用 RGBT 配对加载。
- `rgbt_pair: [visible, infrared]`：把路径中的 `visible` 替换为 `infrared` 来寻找红外图像。
- `names`：类别名称。

如果你的目录名不是 `visible` 和 `infrared`，例如：

```text
YourDataset/
  rgb/
  ir/
```

则 yaml 应写成：

```yaml
rgbt_pair: [rgb, ir]
train: rgb/train
val: rgb/val
```

## 5. 检查 RGB 和红外是否配对

训练前建议先检查配对关系。

默认目录名为 `visible/infrared`：

```bash
python examples/RGBT/check_rgbt_pairs.py D:/datasets/YourDataset/visible/train
python examples/RGBT/check_rgbt_pairs.py D:/datasets/YourDataset/visible/val
```

如果你的目录名是 `rgb/ir`：

```bash
python examples/RGBT/check_rgbt_pairs.py D:/datasets/YourDataset/rgb/train --rgb-token rgb --ir-token ir
```

如果提示 `All infrared pairs were found.`，说明配对正常。

## 6. 可选模型配置

当前提供 8 个 RGBT 模型配置：

```text
ultralytics/cfg/models/v5-RGBT/yolov5-rgbt-earlyfusion.yaml
ultralytics/cfg/models/v5-RGBT/yolov5-rgbt-midfusion.yaml

ultralytics/cfg/models/v8-RGBT/yolov8-rgbt-earlyfusion.yaml
ultralytics/cfg/models/v8-RGBT/yolov8-rgbt-midfusion.yaml

ultralytics/cfg/models/11-RGBT/yolo11-rgbt-earlyfusion.yaml
ultralytics/cfg/models/11-RGBT/yolo11-rgbt-midfusion.yaml

ultralytics/cfg/models/26-RGBT/yolo26-rgbt-earlyfusion.yaml
ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml
```

两种融合方式区别：

- `earlyfusion`：一开始就把 4 通道一起送入主干网络，结构简单，适合作为基线。
- `midfusion`：RGB 和红外分别走分支，在多尺度特征层融合，更符合双模态检测思路。

建议第一次训练先使用：

```text
ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml
```

或者显存较小/想快速测试时使用：

```text
ultralytics/cfg/models/v8-RGBT/yolov8-rgbt-earlyfusion.yaml
```

## 7. 训练自己的数据集

命令行训练：

```bash
yolo detect train model=ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml data=ultralytics/cfg/datasets/your-rgbt.yaml imgsz=640 epochs=100 batch=16
```

使用示例脚本训练：

```bash
python examples/RGBT/train_rgbt.py --model ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml --data ultralytics/cfg/datasets/your-rgbt.yaml --imgsz 640 --epochs 100 --batch 16 --device 0 --name your-rgbt-exp
```

如果显存不足，可以调小：

```bash
--batch 8
```

或：

```bash
--imgsz 512
```

## 8. 验证模型

命令行验证：

```bash
yolo detect val model=runs/rgbt/your-rgbt-exp/weights/best.pt data=ultralytics/cfg/datasets/your-rgbt.yaml imgsz=640 batch=16
```

使用示例脚本验证：

```bash
python examples/RGBT/val_rgbt.py --model runs/rgbt/your-rgbt-exp/weights/best.pt --data ultralytics/cfg/datasets/your-rgbt.yaml --imgsz 640 --batch 16
```

验证阶段会读取 yaml 中的 `rgbt: true` 和 `rgbt_pair`，自动加载红外图像。

## 9. 推理预测

推理时输入路径应指向可见光目录或可见光图像。

命令行推理：

```bash
yolo detect predict model=runs/rgbt/your-rgbt-exp/weights/best.pt source=D:/datasets/YourDataset/visible/test rgbt=True rgbt_pair="[visible,infrared]" imgsz=640 save=True
```

使用示例脚本推理：

```bash
python examples/RGBT/predict_rgbt.py --model runs/rgbt/your-rgbt-exp/weights/best.pt --source D:/datasets/YourDataset/visible/test --imgsz 640
```

如果你的目录名是 `rgb/ir`：

```bash
python examples/RGBT/predict_rgbt.py --model runs/rgbt/your-rgbt-exp/weights/best.pt --source D:/datasets/YourDataset/rgb/test --rgb-token rgb --ir-token ir
```

## 10. 导出模型

导出 ONNX：

```bash
yolo export model=runs/rgbt/your-rgbt-exp/weights/best.pt format=onnx imgsz=640
```

或使用脚本：

```bash
python examples/RGBT/export_rgbt.py --model runs/rgbt/your-rgbt-exp/weights/best.pt --format onnx --imgsz 640
```

注意：导出的模型输入仍然是 4 通道。部署时你需要自己在前处理阶段把 RGB/BGR 图像和红外图像拼成 4 通道，顺序为：

```text
BGR + T
```

## 11. 常见问题

### 11.1 报错找不到红外图像

通常是路径无法替换导致的。检查：

- 可见光路径中是否包含 `visible`。
- 红外目录是否叫 `infrared`。
- 文件名是否完全一致。

如果目录名不同，修改 yaml：

```yaml
rgbt_pair: [rgb, ir]
```

### 11.2 模型输入通道不匹配

确认数据集 yaml 中有：

```yaml
channels: 4
rgbt: true
```

确认模型使用的是 `*-RGBT/*.yaml`，不是普通 YOLO 模型 yaml。

### 11.3 可见光和红外图尺寸不同

代码会把红外图 resize 到可见光图尺寸再拼接。但更推荐你提前做好空间配准和尺寸统一，否则检测效果可能下降。

### 11.4 红外图是彩色伪彩图怎么办

当前代码按灰度方式读取红外图，只取一个红外通道。如果你的红外图是伪彩色，建议先转换成单通道灰度图再训练。

### 11.5 标签应该放 visible 还是 infrared

推荐放在 `visible` 对应路径下。训练列表读取的是可见光路径，标签也按可见光图像路径去找。

## 12. 最小流程总结

1. 整理数据：

```text
YourDataset/visible/train
YourDataset/infrared/train
YourDataset/visible/val
YourDataset/infrared/val
```

2. 写数据集 yaml：

```yaml
path: D:/datasets/YourDataset
train: visible/train
val: visible/val
channels: 4
rgbt: true
rgbt_pair: [visible, infrared]
names:
  0: person
```

3. 检查配对：

```bash
python examples/RGBT/check_rgbt_pairs.py D:/datasets/YourDataset/visible/train
```

4. 训练：

```bash
yolo detect train model=ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml data=ultralytics/cfg/datasets/your-rgbt.yaml imgsz=640 epochs=100 batch=16
```

5. 验证：

```bash
yolo detect val model=runs/detect/train/weights/best.pt data=ultralytics/cfg/datasets/your-rgbt.yaml imgsz=640
```

6. 推理：

```bash
yolo detect predict model=runs/detect/train/weights/best.pt source=D:/datasets/YourDataset/visible/test rgbt=True rgbt_pair="[visible,infrared]" imgsz=640 save=True
```
