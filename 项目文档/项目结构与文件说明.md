## 2. 根目录脚本说明

### 2.1 train_rgbt.py

用于训练 RGB + 红外多模态模型。

常用参数：

```text
--model: 模型 YAML 或权重路径
--data: 数据集 YAML
--imgsz: 图像尺寸
--epochs: 训练轮数
--batch: 批大小
--device: 训练设备
--project: 结果保存目录
--name: 实验名称
```

### 2.2 val_rgbt.py

用于验证训练好的 RGBT 模型。

常用参数：

```text
--model: 训练好的 best.pt 或 last.pt
--data: 数据集 YAML
--split: val/test/train
--imgsz: 图像尺寸
--batch: 批大小
```

### 2.3 predict_rgbt.py

用于对 RGB + 红外图像进行推理。

推理时传入 `visible` 图像目录即可，程序会自动寻找对应红外图像。

常用参数：

```text
--model: 训练好的权重
--source: visible 图像目录
--rgb-token: RGB 目录名，默认 visible
--ir-token: 红外目录名，默认 infrared
--conf: 置信度阈值
```

### 2.4 export_rgbt.py

用于导出模型。

常用参数：

```text
--model: 训练好的权重
--format: 导出格式，如 onnx
--imgsz: 图像尺寸
--dynamic: 是否动态尺寸
--simplify: 是否简化 ONNX
```

### 2.5 check_rgbt_pairs.py

用于检查 `visible` 和 `infrared` 图像是否一一对应。

如果该脚本报缺失，需要先修复数据集再训练。

## 3. 文件夹说明

### 3.1 ultralytics/

核心源码目录，包含数据加载、模型定义、训练引擎、推理引擎和工具函数。

本项目的 RGBT 核心改动主要在这个目录内。

### 3.0 项目文档/

项目中文说明文档目录。

建议新手先阅读：

| 文件 | 作用 |
| --- | --- |
| `README.md` | 文档目录索引 |
| `项目说明文档.md` | 项目技术、算法和整体设计说明 |
| `操作说明文档.md` | 环境配置、数据准备、运行命令和参数说明 |
| `项目结构与文件说明.md` | 项目目录和关键代码文件说明 |
| `前端与结果查看说明.md` | 训练结果、推理结果和 TensorBoard 查看说明 |
| `RGBT网络结构说明.md` | RGBT 网络结构和融合方式说明 |
| `多模态代码迁移教程.md` | RGBT 代码迁移到其他项目的教程 |
| `GitHub发布清单.md` | 开源发布前检查清单 |

### 3.2 ultralytics/data/

数据加载相关代码。

与 RGBT 相关的关键文件：

| 文件 | 作用 |
| --- | --- |
| `base.py` | 基础数据集类，增加了 RGB/红外配对读取和 4 通道拼接逻辑 |
| `dataset.py` | YOLODataset，负责把数据集 YAML 中的 `rgbt` 和 `rgbt_pair` 传给基础数据集 |
| `loaders.py` | 推理数据加载器，支持推理时读取 RGBT 图像 |
| `build.py` | 构建训练/推理数据源，增加了 `rgbt` 参数传递 |

### 3.3 ultralytics/engine/

训练、验证、推理等通用引擎目录。

与 RGBT 相关的关键文件：

| 文件 | 作用 |
| --- | --- |
| `predictor.py` | 推理入口，负责把 `rgbt` 和 `rgbt_pair` 传给推理数据加载器 |

### 3.4 ultralytics/nn/

神经网络模块和模型解析目录。

与 RGBT 相关的关键文件：

| 文件 | 作用 |
| --- | --- |
| `modules/conv.py` | 新增 `ChannelSlice`，用于切分 RGB 和红外通道 |
| `modules/__init__.py` | 导出 `ChannelSlice` 模块 |
| `tasks.py` | 模型构建和 YAML 解析，支持 `channels: 4` 和 `ChannelSlice` |

### 3.5 ultralytics/cfg/

配置文件目录，包含默认参数、模型 YAML 和数据集 YAML。

与 RGBT 相关的关键位置：

| 路径 | 作用 |
| --- | --- |
| `default.yaml` | 默认配置，增加推理用 `rgbt` 和 `rgbt_pair` 参数 |
| `datasets/KAIST8-rgbt.yaml` | KAIST 风格数据集示例 |
| `datasets/rgbt-template.yaml` | 通用 RGBT 数据集模板 |
| `models/26-RGBT/` | YOLO26 RGBT 模型配置 |
| `models/11-RGBT/` | YOLO11 RGBT 模型配置 |
| `models/v8-RGBT/` | YOLOv8 RGBT 模型配置 |
| `models/v5-RGBT/` | YOLOv5 RGBT 模型配置 |

### 3.6 examples/

示例代码目录。

其中 `examples/RGBT/` 保存了 RGBT 示例脚本和说明文档，根目录也放了一份常用脚本，方便新手直接使用。

### 3.7 tests/

测试目录。

与 RGBT 相关的文件：

| 文件 | 作用 |
| --- | --- |
| `test_rgbt.py` | 测试 RGBT 推理加载、训练数据集 4 通道输出、RGBT 模型 YAML 构建 |

### 3.8 docs/

Ultralytics 原项目文档目录。

本项目主要使用根目录中文文档作为 RGBT 使用入口。

### 3.9 runs/

训练、验证、推理结果输出目录。

该目录不建议提交到 GitHub。

### 3.10 docker/

Docker 相关配置目录，来自 Ultralytics 原项目。

如果使用 Docker 部署，可以在此基础上进一步改造。

### 3.11 .github/

GitHub 工作流和 Issue 模板目录，来自 Ultralytics 原项目。

开源发布后可根据需要调整。

### 3.12 .vscode/

VSCode 编辑器配置目录。

个人开发配置不一定需要提交。

## 4. RGBT 模型配置说明

### 4.1 earlyfusion

文件名包含：

```text
earlyfusion
```

表示 RGB 和红外图像在输入阶段拼接为 4 通道。

### 4.2 midfusion

文件名包含：

```text
midfusion
```

表示 RGB 和红外图像先分别进入不同分支，再进行中间层特征融合。
