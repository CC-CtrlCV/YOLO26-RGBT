# 项目说明文档

## 1. 项目简介

本项目是一个基于 Ultralytics YOLO 框架改造的 RGB + 红外多模态目标检测项目，主要面向 RGBT 目标检测任务。

原版 YOLO 通常只处理 3 通道 RGB 图像，本项目增加了 RGB 图像和红外图像的配对读取能力，并支持将二者拼接为 4 通道输入进行训练、验证、推理和模型导出。

## 2. 项目目标

本项目的目标是让使用者可以像训练普通 YOLO 一样训练 RGB + 红外多模态模型。

项目重点解决以下问题：

```text
1. 如何读取一张 RGB 图像和对应红外图像。
2. 如何把 RGB 三通道和红外一通道拼接成 4 通道输入。
3. 如何让 YOLO 模型支持 4 通道输入。
4. 如何构建 RGB/IR 双分支融合网络。
5. 如何让训练、验证、推理、导出流程都支持 RGBT。
6. 如何让新手快速使用自己的数据集训练多模态模型。
```

## 4. 使用的核心算法

本项目以 YOLO 系列目标检测算法为基础，支持 YOLOv5、YOLOv8、YOLOv11、YOLOv26 的 RGBT 模型配置。

项目中涉及的核心算法和结构包括：

```text
YOLO one-stage object detection
Backbone 特征提取网络
Neck 多尺度特征融合网络
Detect 检测头
Early Fusion 早期融合
Mid Fusion 中期融合
ChannelSlice 通道切分模块
RGB/IR 双分支特征提取
```

## 5. RGBT 数据读取算法

本项目训练时以 `visible` 图像路径为入口。

例如：

```text
visible/train/000001.jpg
```

当数据集 YAML 中写有：

```yaml
rgbt: true
rgbt_pair: [visible, infrared]
```

程序会自动把路径中的 `visible` 替换成 `infrared`，从而找到红外图像：

```text
infrared/train/000001.jpg
```

之后程序会读取：

```text
RGB 图像: 3 通道
红外图像: 1 通道
```

并拼接为：

```text
RGBT 图像: 4 通道
```

## 6. Early Fusion

Early fusion 是最简单的多模态融合方式。

它的思路是：

```text
RGB 图像      \
              -> 4 通道输入 -> YOLO Backbone -> Detect
红外图像     /
```

优点是结构简单、容易跑通、适合作为 baseline。

缺点是 RGB 和红外信息从输入层就混合在一起，没有显式建模两个模态之间的差异。

## 7. Mid Fusion

Mid fusion 会先把 RGB 和红外分成两个分支。

它的思路是：

```text
RGB 图像 -> RGB 分支 \
                    -> 特征融合 -> YOLO Head -> Detect
红外图像 -> IR 分支  /
```

本项目通过 `ChannelSlice` 从 4 通道输入中拆出两个分支：

```text
ChannelSlice [0, 3] -> RGB 三通道
ChannelSlice [3, 4] -> 红外一通道
```

优点是更符合多模态检测的常见设计，便于后续加入注意力融合、加权融合或 Transformer 融合模块。

缺点是网络更复杂，显存占用和训练难度可能更高。

## 9. 项目特色

本项目的主要特色是：

```text
1. 支持 RGB + 红外图像自动配对。
2. 支持 4 通道 RGBT 输入。
3. 支持 YOLOv5、YOLOv8、YOLOv11、YOLOv26 多版本配置。
4. 支持 early fusion 和 mid fusion 两种结构。

7. 适合新手快速运行，也适合研究者二次开发。
```
