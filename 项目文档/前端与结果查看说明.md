# 前端与结果查看说明

本项目本身不是 Web 前端项目，没有单独的网页后台或 Vue/React 页面。

这里的“前端页面”主要指 Ultralytics 训练、验证、推理后生成的可视化结果，包括预测图片、训练曲线、指标图、TensorBoard 页面等。

## 1. 训练结果在哪里看

训练完成后，默认结果保存在：

```text
runs/rgbt/实验名称/
```

例如：

```text
runs/rgbt/yolo26-rgbt-midfusion/
```

常见文件：

| 文件或目录             | 作用                  |
| ---------------------- | --------------------- |
| `weights/best.pt`      | 验证集效果最好的权重  |
| `weights/last.pt`      | 最后一轮训练权重      |
| `results.csv`          | 每轮训练指标数据      |
| `results.png`          | 训练损失和指标曲线    |
| `confusion_matrix.png` | 混淆矩阵              |
| `PR_curve.png`         | Precision-Recall 曲线 |
| `F1_curve.png`         | F1 曲线               |
| `P_curve.png`          | Precision 曲线        |
| `R_curve.png`          | Recall 曲线           |
| `train_batch*.jpg`     | 训练 batch 可视化图   |
| `val_batch*.jpg`       | 验证 batch 可视化图   |

## 2. 怎么判断训练是否正常

可以重点看：

```text
1. results.png 中 loss 是否整体下降。
2. mAP50 和 mAP50-95 是否上升。
3. train_batch*.jpg 中标签是否画在正确目标上。
4. val_batch*.jpg 中预测框是否接近真实目标。
5. confusion_matrix.png 中类别是否混淆严重。
```

如果 loss 不下降或预测框乱飞，通常需要检查：

```text
数据标注是否正确
RGB 和红外是否配对
类别 names 是否正确
图像尺寸是否一致
学习率和 batch 是否合适
```

## 3. 推理结果在哪里看

运行：

```powershell
python predict_rgbt.py `
  --model runs/rgbt/yolo26-rgbt-midfusion/weights/best.pt `
  --source D:/datasets/YourDataset/visible/val `
  --imgsz 640
```

预测结果默认保存在：

```text
runs/rgbt-predict/实验名称/
```

例如：

```text
runs/rgbt-predict/yolo26-rgbt-midfusion/
```

里面会保存画好检测框的图片。

## 4. 推理时怎么看 RGB 和红外是否真的一起用了

推理命令中的 `--source` 只传入 visible 目录：

```text
D:/datasets/YourDataset/visible/val
```

代码会根据：

```text
--rgb-token visible
--ir-token infrared
```

自动寻找：

```text
D:/datasets/YourDataset/infrared/val
```

如果红外图像缺失，程序会报错。

因此，只要推理可以正常完成，并且使用的是 RGBT 模型和 `predict_rgbt.py`，就说明 RGB 和红外配对读取流程已经生效。

## 5. TensorBoard 可视化

如果安装了 TensorBoard，可以运行：

```powershell
tensorboard --logdir runs
```

然后在浏览器中打开：

```text
http://localhost:6006
```

TensorBoard 中可以查看：

```text
训练 loss
验证指标
mAP 曲线
学习率变化
```

如果浏览器打不开，检查：

```text
1. tensorboard 是否安装。
2. 命令是否在项目根目录运行。
3. 端口 6006 是否被占用。
```

## 6. 没有 Web 页面怎么办

这是正常的。

本项目主要是目标检测训练项目，不是 Web 应用。

结果查看主要依赖：

```text
runs 目录中的图片和曲线
命令行输出的指标
TensorBoard 可视化页面
```

如果后续需要真正的 Web 前端，可以基于以下方式扩展：

```text
Streamlit
Gradio
Flask + HTML
FastAPI + Vue/React
```

## 7. 新手推荐查看顺序

训练后建议按这个顺序看：

```text
1. runs/rgbt/实验名称/results.png
2. runs/rgbt/实验名称/train_batch*.jpg
3. runs/rgbt/实验名称/val_batch*.jpg
4. runs/rgbt/实验名称/weights/best.pt
5. runs/rgbt-predict/实验名称/预测图片
```

如果这些文件都能正常生成，说明项目训练和推理流程基本跑通。
