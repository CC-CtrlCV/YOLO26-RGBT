# 操作说明文档

本文档说明如何配置环境、准备数据、修改参数、运行训练、验证、推理和导出。

## 1. 环境要求

推荐环境：

```text
Windows 10/11 或 Linux
Python 3.8+
PyTorch
CUDA GPU，推荐但不是必须
OpenCV
Ultralytics 相关依赖
```

如果使用 GPU，请先确认显卡驱动、CUDA 和 PyTorch 版本匹配。

## 2. 安装依赖

进入项目根目录：

```powershell
cd D:\XM\DMT\ultralytics-main
```

安装基础依赖：

```bash
pip install -r requirements.txt
```

以本地源码方式安装项目：

```bash
pip install -e .
```

如果 `torch` 安装失败，建议去 PyTorch 官网根据自己的 CUDA 版本复制安装命令。

## 3. 数据集应该放在哪里

数据集可以放在任意磁盘位置，只要在 YAML 中写对路径即可。

推荐结构：

```text
D:/datasets/YourDataset/
├── visible/
│   ├── train/
│   │   ├── 000001.jpg
│   │   └── 000002.jpg
│   └── val/
│       ├── 000101.jpg
│       └── 000102.jpg
├── infrared/
│   ├── train/
│   │   ├── 000001.jpg
│   │   └── 000002.jpg
│   └── val/
│       ├── 000101.jpg
│       └── 000102.jpg
└── labels/
    ├── train/
    │   ├── 000001.txt
    │   └── 000002.txt
    └── val/
        ├── 000101.txt
        └── 000102.txt
```

注意事项：

```text
1. visible 和 infrared 中的图像文件名必须一一对应。
2. RGB 图像和红外图像尺寸最好一致。
3. 标签使用 YOLO 格式。
4. 标签通常与 visible 图像同名。
```

YOLO 标签格式：

```text
class_id x_center y_center width height
```

坐标必须是归一化后的 0 到 1 之间的数值。

## 4. 修改数据集 YAML

建议复制模板：

```text
ultralytics/cfg/datasets/rgbt-template.yaml
```

例如新建：

```text
ultralytics/cfg/datasets/my-rgbt.yaml
```

内容示例：

```yaml
path: D:/datasets/YourDataset
train: visible/train
val: visible/val
test:

channels: 4
rgbt: true
rgbt_pair: [visible, infrared]

names:
  0: person
  1: car
```

需要修改的参数：

```text
path: 数据集根目录
train: 训练集 visible 路径
val: 验证集 visible 路径
names: 自己的数据集类别
```

如果你的目录名不是 `visible` 和 `infrared`，例如是 `rgb` 和 `ir`，则写成：

```yaml
rgbt_pair: [rgb, ir]
```

## 5. 检查 RGB 和红外图像配对

默认目录名是 `visible` 和 `infrared`：

```powershell
python check_rgbt_pairs.py D:/datasets/YourDataset/visible/train
python check_rgbt_pairs.py D:/datasets/YourDataset/visible/val
```

如果目录名是 `rgb` 和 `ir`：

```powershell
python check_rgbt_pairs.py D:/datasets/YourDataset/rgb/train --rgb-token rgb --ir-token ir
```

如果出现缺失提示，需要补齐对应红外图像，或者检查文件名是否一致。

## 6. 选择模型 YAML

推荐先用 YOLO26-RGBT：

```text
ultralytics/cfg/models/26-RGBT/yolo26-rgbt-earlyfusion.yaml
ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml
```

快速跑通推荐：

```text
yolo26-rgbt-earlyfusion.yaml
```

正式多模态实验推荐：

```text
yolo26-rgbt-midfusion.yaml
```

## 7. Windows 命令写法

很多教程里的反斜杠换行：

```bash
python train_rgbt.py \
  --model xxx \
  --data xxx
```

这是 Linux/macOS 写法，Windows PowerShell 不能这样直接用。

PowerShell 应该使用反引号：

```powershell
python train_rgbt.py `
  --model ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml `
  --data ultralytics/cfg/datasets/my-rgbt.yaml `
  --imgsz 640 `
  --epochs 100 `
  --batch 16 `
  --device 0 `
  --name yolo26-rgbt-midfusion
```

Windows CMD 或者最稳妥方式是写成一行：

```cmd
python train_rgbt.py --model ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml --data ultralytics/cfg/datasets/my-rgbt.yaml --imgsz 640 --epochs 100 --batch 16 --device 0 --name yolo26-rgbt-midfusion
```

## 8. 开始训练

PowerShell：

```powershell
python train_rgbt.py `
  --model ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml `
  --data ultralytics/cfg/datasets/my-rgbt.yaml `
  --imgsz 640 `
  --epochs 100 `
  --batch 16 `
  --device 0 `
  --name yolo26-rgbt-midfusion
```

参数说明：

```text
--model: 模型结构 YAML 或权重路径
--data: 数据集 YAML
--imgsz: 输入图像尺寸
--epochs: 训练轮数
--batch: 批大小
--device: 训练设备，0 表示第 0 张 GPU，cpu 表示 CPU
--name: 实验名称
```

训练结果默认保存到：

```text
runs/rgbt/yolo26-rgbt-midfusion/
```

## 9. 验证模型

```powershell
python val_rgbt.py `
  --model runs/rgbt/yolo26-rgbt-midfusion/weights/best.pt `
  --data ultralytics/cfg/datasets/my-rgbt.yaml `
  --imgsz 640 `
  --batch 16 `
  --device 0
```

验证结果会输出 mAP、Precision、Recall 等指标。

## 10. 推理预测

```powershell
python predict_rgbt.py `
  --model runs/rgbt/yolo26-rgbt-midfusion/weights/best.pt `
  --source D:/datasets/YourDataset/visible/val `
  --imgsz 640
```

这里只需要传入 `visible` 图像目录。

程序会根据：

```text
rgbt_pair: [visible, infrared]
```

自动找到对应的红外图像。

预测结果默认保存到：

```text
runs/rgbt-predict/
```

## 11. 导出模型

导出 ONNX：

```powershell
python export_rgbt.py `
  --model runs/rgbt/yolo26-rgbt-midfusion/weights/best.pt `
  --format onnx `
  --imgsz 640
```

常见导出格式：

```text
onnx
torchscript
engine
openvino
```

不同格式可能需要额外依赖，具体以 Ultralytics 原框架要求为准。

## 12. 常见问题

### 12.1 找不到红外图像

检查：

```text
visible 和 infrared 目录名是否一致
文件名是否一致
rgbt_pair 是否写对
```

### 12.2 输入通道不匹配

检查数据集 YAML 是否写了：

```yaml
channels: 4
rgbt: true
```

检查模型 YAML 是否是 RGBT 版本。

### 12.3 Windows 换行命令报错

Windows PowerShell 使用反引号，不使用 Linux 的反斜杠。

### 12.4 显存不够

可以减小：

```text
--batch
--imgsz
```

也可以先使用 earlyfusion 模型。
