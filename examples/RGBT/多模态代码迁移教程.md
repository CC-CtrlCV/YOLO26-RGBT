# RGBT 多模态代码迁移教程

本文档面向刚下载并跑通本项目的用户，说明如何把本项目中的 RGB + 红外双模态能力迁移到自己的 YOLO/Ultralytics 代码包中。

目标是把下面这套能力搬过去：

- 训练时自动读取 `visible` 图像和对应 `infrared` 图像。
- 验证时自动读取 RGB + IR 双模态输入。
- 推理时通过 `rgbt=True` 自动配对红外图像。
- 模型 yaml 支持 4 通道输入。
- 模型 yaml 支持 `earlyfusion` 和 `midfusion` 两类结构。

本文档以“你的代码包也是 Ultralytics 风格项目”为前提。如果你的项目不是 Ultralytics 结构，也可以参考核心逻辑手动适配。

## 1. 迁移前先确认你的项目结构

你的项目中通常应该有类似目录：

```text
YourProject/
  ultralytics/
    cfg/
      default.yaml
      datasets/
      models/
    data/
      base.py
      build.py
      dataset.py
      loaders.py
    engine/
      predictor.py
    nn/
      modules/
        __init__.py
        conv.py
      tasks.py
```

如果你的项目没有这些文件，说明代码结构差异较大，不建议直接复制粘贴，应先理解第 3 节的核心逻辑后手动改。

## 2. 推荐迁移顺序

建议按以下顺序迁移：

1. 迁移数据加载逻辑。
2. 迁移推理入口参数。
3. 迁移网络模块 `ChannelSlice`。
4. 迁移 RGBT 模型 yaml。
5. 迁移数据集 yaml 模板。
6. 迁移 examples 脚本。
7. 做最小验证。

不要一开始就把整个项目全量覆盖到自己的项目里。这样最容易把原项目已有功能弄乱。

## 3. 核心原理

### 3.1 数据层核心逻辑

本项目不要求你提前生成 4 通道图像，而是在读取图像时动态合成：

```text
visible/img001.jpg    -> BGR 3通道
infrared/img001.jpg   -> T 1通道

合成：
BGR + T -> 4通道 RGBT
```

路径配对规则由 `rgbt_pair` 控制：

```yaml
rgbt_pair: [visible, infrared]
```

意思是：

```python
infrared_path = visible_path.replace("visible", "infrared", 1)
```

### 3.2 模型层核心逻辑

模型 yaml 里声明：

```yaml
channels: 4
```

表示模型输入是 4 通道。

`earlyfusion` 直接把 4 通道一起送进 backbone。

`midfusion` 使用 `ChannelSlice` 切通道：

```yaml
- [0, 1, ChannelSlice, [0, 3]] # RGB/BGR
- [0, 1, ChannelSlice, [3, 4]] # infrared
```

然后 RGB 和 IR 分别提特征，再在 P3/P4/P5 特征层融合。

## 4. 必须迁移的代码文件

以下文件是最小必需迁移范围。

### 4.1 数据集训练/验证加载

源文件：

```text
ultralytics/data/base.py
ultralytics/data/dataset.py
```

需要迁移的能力：

- `BaseDataset.__init__` 增加参数：

```python
rgbt: bool = False
rgbt_pair: list[str] | tuple[str, str] = ("visible", "infrared")
```

- 增加属性：

```python
self.rgbt = bool(rgbt)
self.rgbt_pair = self._validate_rgbt_pair(rgbt_pair)
```

- 增加 `_read_image()`，在 `rgbt=True` 时读取并拼接双模态图像。
- `YOLODataset` 初始化时从 `data.yaml` 读取：

```python
rgbt = self.data.get("rgbt", False)
rgbt_pair = self.data.get("rgbt_pair", ("visible", "infrared"))
```

迁移后，训练和验证阶段才能自动读取红外图像。

### 4.2 推理加载

源文件：

```text
ultralytics/data/loaders.py
ultralytics/data/build.py
ultralytics/engine/predictor.py
```

需要迁移的能力：

- `LoadImagesAndVideos` 增加：

```python
rgbt = False
rgbt_pair = ("visible", "infrared")
```

- 增加 `_read_rgbt_image()`。
- `load_inference_source()` 增加：

```python
rgbt = False
rgbt_pair = ("visible", "infrared")
```

- `predictor.py` 中调用 `load_inference_source()` 时传入：

```python
rgbt = self.args.rgbt
rgbt_pair = self.args.rgbt_pair
```

迁移后，推理时才能这样使用：

```bash
yolo detect predict model=best.pt source=YourDataset/visible/test rgbt=True rgbt_pair="[visible,infrared]"
```

### 4.3 默认配置参数

源文件：

```text
ultralytics/cfg/default.yaml
```

增加：

```yaml
rgbt: False
rgbt_pair: [visible, infrared]
```

如果不加这个，CLI 或 Python 参数中传 `rgbt=True` 时可能会报参数不存在。

### 4.4 网络通道切片模块

源文件：

```text
ultralytics/nn/modules/conv.py
ultralytics/nn/modules/__init__.py
ultralytics/nn/tasks.py
```

需要迁移：

在 `conv.py` 中增加：

```python
class ChannelSlice(nn.Module):
    def __init__(self, start: int, end: int):
        super().__init__()
        self.start = start
        self.end = end

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return x[:, self.start : self.end]
```

在 `__init__.py` 中导入并加入 `__all__`。

在 `tasks.py` 中导入 `ChannelSlice`，并在 `parse_model()` 中增加：

```python
elif m is ChannelSlice:
    c2 = args[1] - args[0]
```

还要确认模型构造时尊重 yaml 中的 `channels`：

```python
ch = self.yaml["channels"] = self.yaml.get("channels", ch)
```

否则直接 `YOLO("xxx-rgbt.yaml")` 时可能仍按 3 通道构建。

## 5. 必须迁移的配置文件

### 5.1 数据集模板

建议复制：

```text
ultralytics/cfg/datasets/rgbt-template.yaml
```

模板内容大致为：

```yaml
path: /path/to/rgbt-dataset
train: visible/train
val: visible/val
test:

channels: 4
rgbt: true
rgbt_pair: [visible, infrared]

names:
  0: person
```

你可以按自己的数据集类别修改 `names`。

### 5.2 模型 yaml

建议至少迁移一组模型配置：

```text
ultralytics/cfg/models/26-RGBT/yolo26-rgbt-earlyfusion.yaml
ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml
```

如果你的项目只需要 YOLOv8，则迁移：

```text
ultralytics/cfg/models/v8-RGBT/yolov8-rgbt-earlyfusion.yaml
ultralytics/cfg/models/v8-RGBT/yolov8-rgbt-midfusion.yaml
```

本项目当前提供：

```text
ultralytics/cfg/models/v5-RGBT/
ultralytics/cfg/models/v8-RGBT/
ultralytics/cfg/models/11-RGBT/
ultralytics/cfg/models/26-RGBT/
```

## 6. 建议迁移的 examples 脚本

建议复制整个目录：

```text
examples/RGBT/
```

其中包括：

```text
train_rgbt.py
val_rgbt.py
predict_rgbt.py
export_rgbt.py
check_rgbt_pairs.py
README.md
使用说明书.md
多模态代码迁移教程.md
```

这些脚本不是核心框架必需，但对新用户非常重要。

## 7. 迁移后的数据集结构

你的数据集应整理成：

```text
YourDataset/
  visible/
    train/
      img001.jpg
      img001.txt
    val/
      img101.jpg
      img101.txt
    test/
      img201.jpg

  infrared/
    train/
      img001.jpg
    val/
      img101.jpg
    test/
      img201.jpg
```

标签推荐放在 `visible` 对应目录下。

可见光和红外必须一一对应：

```text
visible/train/img001.jpg
infrared/train/img001.jpg
```

## 8. 迁移后的最小验证

### 8.1 检查模型能否构建

进入你的项目根目录：

```bash
python -c "from ultralytics import YOLO; YOLO('ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml'); print('ok')"
```

如果输出 `ok`，说明模型 yaml、`ChannelSlice` 和 `channels: 4` 基本正常。

### 8.2 检查数据是否配对

```bash
python examples/RGBT/check_rgbt_pairs.py D:/datasets/YourDataset/visible/train
```

正常输出：

```text
Checked xxx visible images.
All infrared pairs were found.
```

### 8.3 跑一个很小的训练

```bash
yolo detect train model=ultralytics/cfg/models/26-RGBT/yolo26-rgbt-midfusion.yaml data=ultralytics/cfg/datasets/your-rgbt.yaml imgsz=320 epochs=1 batch=2
```

第一次迁移不要直接跑大训练，先用小图、小 batch、1 epoch 验证流程。

### 8.4 跑验证

```bash
yolo detect val model=runs/detect/train/weights/best.pt data=ultralytics/cfg/datasets/your-rgbt.yaml imgsz=320 batch=2
```

### 8.5 跑推理

```bash
yolo detect predict model=runs/detect/train/weights/best.pt source=D:/datasets/YourDataset/visible/test rgbt=True rgbt_pair="[visible,infrared]" imgsz=320 save=True
```

## 9. 迁移时不要做的事

不建议一开始做这些：

1. 不要整包覆盖自己的项目。
2. 不要把 YOLOv5、v8、11、26 的所有 yaml 一次性混到原模型目录里。
3. 不要先改损失函数、验证指标、NMS。
4. 不要先迁移多光谱 7 通道、10 通道、16bit 逻辑。
5. 不要先迁移复杂融合模块，例如 TransformerFusion、MCF、CTF、PGI。

第一版只保证 RGB + 红外 4 通道检测闭环即可。

## 10. 常见迁移问题

### 10.1 报错：找不到参数 `rgbt`

说明你没有改：

```text
ultralytics/cfg/default.yaml
```

需要增加：

```yaml
rgbt: False
rgbt_pair: [visible, infrared]
```

### 10.2 报错：`ChannelSlice` 不存在

说明你没有正确修改：

```text
ultralytics/nn/modules/conv.py
ultralytics/nn/modules/__init__.py
ultralytics/nn/tasks.py
```

### 10.3 报错：模型需要 4 通道，但输入是 3 通道

通常是以下原因：

- 数据集 yaml 没有 `channels: 4`。
- 数据集 yaml 没有 `rgbt: true`。
- 使用了普通 YOLO yaml，而不是 RGBT yaml。
- `DetectionModel` 没有读取 yaml 中的 `channels`。

### 10.4 报错：红外图找不到

检查：

- 可见光路径中是否包含 `visible`。
- 红外目录是否叫 `infrared`。
- 文件名是否完全一致。
- 如果目录名不同，是否正确设置 `rgbt_pair`。

### 10.5 推理能跑，训练不能跑

说明你只迁移了 `loaders.py` 推理逻辑，没有迁移 `base.py` 和 `dataset.py` 训练逻辑。

### 10.6 训练能跑，推理不能跑

说明你迁移了 `base.py` 和 `dataset.py`，但没有迁移：

```text
ultralytics/data/loaders.py
ultralytics/data/build.py
ultralytics/engine/predictor.py
```

## 11. 建议的迁移检查清单

迁移完成后逐项确认：

```text
[ ] default.yaml 中有 rgbt 和 rgbt_pair
[ ] BaseDataset 支持 rgbt 和 rgbt_pair
[ ] YOLODataset 从 data.yaml 读取 rgbt 配置
[ ] LoadImagesAndVideos 支持 RGBT 推理
[ ] load_inference_source 支持 rgbt 参数
[ ] predictor.py 传入 rgbt 参数
[ ] conv.py 中有 ChannelSlice
[ ] modules/__init__.py 导出了 ChannelSlice
[ ] tasks.py 能解析 ChannelSlice
[ ] DetectionModel 使用 yaml 中的 channels
[ ] 至少有一个 RGBT 模型 yaml
[ ] 至少有一个 RGBT 数据集 yaml
[ ] 配对检查脚本能跑
[ ] 1 epoch 小训练能跑
[ ] val 能跑
[ ] predict 能跑
```
